Boost Critical Paths with JSI TurboModules in React Native

📅 2026-06-17
Learn how JSI TurboModules let you run native code on the JavaScript thread without the bridge overhead, dramatically speeding up performance‑critical operations.
performance jsi react-native

Insight

React Native’s classic NativeModules communicate over a serialized bridge, which adds latency for tight loops or high‑frequency calls. The JavaScript Interface (JSI) removes that barrier by exposing native functions directly to the JavaScript engine. When you pair JSI with TurboModules, the native implementation is instantiated once and stays resident, letting you call it like a regular JS function. This is ideal for cryptography, image processing, or any CPU‑heavy task that would otherwise stall the UI.

Example

// src/native/MathTurboModule.ts
import {TurboModuleRegistry} from 'react-native';

export interface MathSpec {
  add(a: number, b: number): number;
  multiply(a: number, b: number): number;
}

const MathTurbo = TurboModuleRegistry.getEnforcing<MathSpec>('MathTurbo');

export const add = (a: number, b: number) => MathTurbo.add(a, b);
export const multiply = (a: number, b: number) => MathTurbo.multiply(a, b);

The native side is written in C++/Java and registered as MathTurbo. The JS wrapper above feels like a normal function call—no async bridge, no JSON marshaling.

Takeaway

If a function runs many times per frame, replace its NativeModules call with a JSI TurboModule. You’ll see lower latency and smoother UI, especially on low‑end devices. Start by profiling hot paths, then migrate only those critical pieces to JSI for maximum impact.
