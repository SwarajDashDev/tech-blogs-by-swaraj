Extend Native Modules in Expo with Config Plugins (No Eject Needed)

📅 2026-07-24
Learn how to add custom native code to an Expo-managed app using config plugins, avoiding the heavy‑weight eject process.
expo config-plugins react-native

Insight

Many developers assume that adding a native library to an Expo project forces a full eject to the bare workflow. Since Expo SDK 48, config plugins let you inject native changes at build time while staying in the managed workflow. The plugin runs during eas build (or expo prebuild) and modifies the generated Android and iOS projects, so you get the same capabilities as a bare app without the maintenance overhead.

Example

// plugins/withCustomFont.ts
import { ConfigPlugin, AndroidConfig, IOSConfig } from '@expo/config-plugins';

const withCustomFont: ConfigPlugin = (config) => {
  // Android: add a font asset to app/src/main/assets/fonts
  config = AndroidConfig.Paths.withGradleProperties(config, (props) => {
    props['android.enableAapt2'] = true;
    return props;
  });
  // iOS: add a font to Info.plist
  config = IOSConfig.Fonts.addFont(config, {
    path: './assets/fonts/Inter-Regular.ttf',
    name: 'Inter-Regular',
  });
  return config;
};
export default withCustomFont;

Add the plugin to app.json:

{
  "expo": {
    "plugins": ["./plugins/withCustomFont"]
  }
}

Takeaway

Use a tiny config‑plugin to hook into the native project generation step. This keeps your workflow lightweight, lets you version‑control native tweaks, and avoids the pain of a full eject.
