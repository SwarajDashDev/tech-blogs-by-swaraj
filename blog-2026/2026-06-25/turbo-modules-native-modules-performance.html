TurboModules: Boosting Native Bridge Performance in React Native

📅 2026-06-25
Learn how React Native's TurboModules reduce bridge overhead and improve UI latency by exposing native APIs as JavaScript‑typed modules.
react native turbo modules performance

Insight

React Native 0.74 introduced TurboModules, a new native module system that replaces the classic bridge with a JSI‑based, type‑safe interface. Unlike the legacy bridge, TurboModules are loaded lazily and communicate directly with JavaScript via shared memory, cutting down serialization costs and eliminating the main‑thread bottleneck. This translates to smoother animations and faster data‑heavy interactions, especially on low‑end devices.

Example

// MyTurboModule.ts
import { TurboModule, TurboModuleRegistry } from 'react-native';

export interface Spec extends TurboModule {
  getDeviceInfo(): Promise<string>;
}

export default (TurboModuleRegistry.getEnforcing<Spec>('MyTurboModule') as Spec);

// MyTurboModule.java (Android)
@ReactModule(name = MyTurboModule.NAME)
public class MyTurboModule extends TurboReactModule {
  public static final String NAME = "MyTurboModule";
  public MyTurboModule(ReactApplicationContext context) { super(context); }
  @ReactMethod
  public void getDeviceInfo(Promise promise) {
    promise.resolve(Build.MODEL);
  }
}

Takeaway

When performance matters, migrate hot paths to TurboModules. Start by exposing a single native method, verify the latency drop, then incrementally refactor the rest of your native bridge code.
