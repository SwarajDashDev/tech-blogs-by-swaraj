Monorepo with Yarn Workspaces for Shared React Native UI

📅 2026-07-14
Learn how Yarn Workspaces let you keep a single repository for native and web UI components, reducing duplication and easing maintenance.
monorepo yarn react native

Insight

A monorepo backed by Yarn Workspaces lets you house a shared UI library alongside your iOS, Android, and Expo projects. Because all packages resolve from the same node_modules tree, you avoid version drift and can import the same TypeScript components without extra bundling steps. This pattern also enables atomic releases: bump the UI package version once and every app instantly gets the update.

Example

// root package.json
{
  "private": true,
  "workspaces": ["packages/*", "apps/*"]
}

// packages/ui/Button.tsx
export const Button = ({title, onPress}: {title:string; onPress:()=>void}) => (
  <TouchableOpacity onPress={onPress} style={styles.btn}>
    <Text>{title}</Text>
  </TouchableOpacity>
);

// apps/mobile/App.tsx
import {Button} from "@myorg/ui";

export default function App() {
  return <Button title="Tap me" onPress={() => console.log('pressed')} />;
}

Takeaway

Keep UI code in a dedicated workspace (e.g., packages/ui) and reference it with a scoped import. When you add a new component, it instantly becomes available to all apps, cutting down on copy‑paste bugs and keeping your design system consistent.
