Sharing UI Components Across React Native & Next.js with a Monorepo

📅 2026-07-22
Learn how to structure a Yarn workspaces monorepo so you can write a single UI component library in TypeScript and reuse it in both React Native and Next.js, keeping styling consistent and build times low.
monorepo typescript react native

Insight

A single source of truth for UI components eliminates visual drift between your mobile app and its web companion. By placing a ui package at the root of a Yarn workspaces monorepo and configuring TypeScript path aliases, you can import the same component in a React Native project and a Next.js app. The key is to keep platform‑specific code isolated behind react-native and web entry points, while sharing the core logic and styling tokens.

Example

// packages/ui/src/Button.tsx
import { Platform, TouchableOpacity, Text } from 'react-native';

export const Button = ({ title, onPress }: { title: string; onPress: () => void }) => (
  <TouchableOpacity
    onPress={onPress}
    style={{
      backgroundColor: Platform.OS === 'web' ? '#0066ff' : '#0a84ff',
      padding: 12,
      borderRadius: 4,
    }}
  >
    <Text style={{ color: '#fff' }}>{title}</Text>
  </TouchableOpacity>
);

// tsconfig.json (root)
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": { "@ui/*": ["packages/ui/src/*"] }
  }
}

// apps/native/App.tsx
import { Button } from '@ui/Button';

// apps/web/pages/index.tsx
import { Button } from '@ui/Button';

Takeaway

Set up a Yarn workspace with a shared ui package, expose it via a TypeScript path alias, and let each platform resolve its own implementation details. This gives you one component file, one source of truth, and faster iteration across both mobile and web.
