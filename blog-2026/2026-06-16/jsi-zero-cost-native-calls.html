Zero‑Cost Native Calls with JSI in React Native

📅 2026-06-16
Learn how JavaScript Interface (JSI) lets you call native code without the bridge overhead, delivering near‑native performance for hot paths.
react native jsi performance

Insight

React Native’s classic bridge serializes data between JavaScript and native threads, which can become a bottleneck for high‑frequency operations like cryptography or image processing. The JavaScript Interface (JSI) replaces the bridge with a shared memory model, allowing JavaScript to invoke native functions directly on the UI or JS thread. Because calls are zero‑copy and synchronous, you get deterministic latency and can keep heavy work off the bridge entirely.

Example

// native/MathModule.cpp (C++)
#include <jsi/jsi.h>
using namespace facebook::jsi;

void installMath(ModuleRegistry& registry) {
  auto math = Function::createFromHostFunction(
    runtime, "add", 2,
    [](Runtime& rt, const Value&, const Value* args, size_t) {
      return Value(args[0].asNumber() + args[1].asNumber());
    });
  rt.global().setProperty(rt, "add", std::move(math));
}

// App.tsx (JS)
import { useEffect } from 'react';
declare const add: (a: number, b: number) => number;

export default function App() {
  useEffect(() => {
    console.log('JSI add:', add(3, 4)); // → 7, no bridge hop
  }, []);
  return null;
}

Takeaway

When you need sub‑millisecond native calls, expose the function via JSI instead of the bridge. A tiny native shim plus a global JS function gives you native‑level speed without rewriting your whole app architecture.
