Running Periodic Background Tasks in Expo

📅 2026-06-28
Learn how to schedule reliable background work in Expo apps using expo-task-manager and expo-background-fetch, keeping your app data fresh even when closed.
expo background react-native

Insight

Expo provides a lightweight way to run code while your app is in the background using expo-task-manager together with expo-background-fetch. Unlike push notifications, background fetch lets you periodically pull new data, clean caches, or sync local changes without user interaction. The system decides the exact schedule based on battery and network conditions, but you can hint at frequency and required network type. This approach works on both iOS and Android and requires no native code, making it perfect for OTA‑updatable projects.

Example

import * as TaskManager from 'expo-task-manager';
import * as BackgroundFetch from 'expo-background-fetch';

const FETCH_TASK = 'background-sync';

TaskManager.defineTask(FETCH_TASK, async () => {
  // Perform your sync logic here
  await fetchUpdates();
  return BackgroundFetch.Result.NewData;
});

await BackgroundFetch.registerTaskAsync(FETCH_TASK, {
  minimumInterval: 15 * 60, // 15 minutes
  stopOnTerminate: false,
  startOnBoot: true,
});

Takeaway

Register a background fetch task early (e.g., in App.tsx) and keep the work lightweight. Test on real devices, as simulators often skip background execution. By leveraging Expo's managed APIs, you can keep your app data up‑to‑date without writing native modules.
