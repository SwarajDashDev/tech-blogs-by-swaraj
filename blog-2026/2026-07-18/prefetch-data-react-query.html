Prefetch Data with React Query to Speed Up Screen Transitions

📅 2026-07-18
Leverage React Query's prefetch capabilities to load data in the background before a user navigates, resulting in near‑instant screen renders.
react-query performance react-native

Insight

When navigating between screens, the UI often stalls while the next view fetches its data. React Query’s prefetchQuery lets you load that data ahead of time—for example, when a list item becomes visible. By caching the result before the navigation occurs, the destination screen can read from the cache instantly, eliminating the loading spinner and improving perceived performance.

Example

import { useNavigation } from '@react-navigation/native';
import { useQueryClient, useQuery } from '@tanstack/react-query';
import { TouchableOpacity, Text } from 'react-native';

function UserItem({ id }: { id: string }) {
  const navigation = useNavigation();
  const queryClient = useQueryClient();

  // Prefetch user details when the item is pressed (but before navigation)
  const handlePress = async () => {
    await queryClient.prefetchQuery(['user', id], () => fetchUser(id));
    navigation.navigate('UserDetail', { id });
  };

  return (
    <TouchableOpacity onPress={handlePress}>
      <Text>Open User {id}</Text>
    </TouchableOpacity>
  );
}

function UserDetailScreen({ route }: any) {
  const { id } = route.params;
  const { data } = useQuery(['user', id], () => fetchUser(id));
  return <Text>{data?.name ?? 'Loading...'}</Text>;
}

Takeaway

Trigger prefetchQuery just before navigation (e.g., on press or hover). The next screen will read the cached data instantly, delivering a smoother, more responsive user experience without extra boiler‑side code.
