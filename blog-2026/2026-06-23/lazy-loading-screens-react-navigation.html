Lazy‑Loading Screens with React Navigation and Suspense

📅 2026-06-23
Speed up initial bundle size by deferring screen code until navigation, using React.lazy and the new Suspense support in React Navigation.
react navigation performance lazy loading

Insight

React Navigation 6+ can now work with React's native Suspense to lazy‑load screen components. By splitting each screen into its own chunk, the initial bundle stays tiny, which improves cold‑start time on both iOS and Android. This pattern is especially useful for large apps where many screens are rarely visited.

Example

// routes/HomeScreen.tsx
export default function HomeScreen() {
  return <View><Text>Home</Text></View>;
}

// routes/SettingsScreen.tsx (lazy loaded)
import { lazy } from 'react';
const SettingsScreen = lazy(() => import('./SettingsScreen'));

// AppNavigator.tsx
import { NavigationContainer } from '@react-navigation/native';
import { createNativeStackNavigator } from '@react-navigation/native-stack';
import { Suspense } from 'react';
import HomeScreen from './routes/HomeScreen';

const Stack = createNativeStackNavigator();

export default function AppNavigator() {
  return (
    <NavigationContainer>
      <Stack.Navigator>
        <Stack.Screen name="Home" component={HomeScreen} />
        <Stack.Screen
          name="Settings"
          // @ts-ignore – React Navigation expects a component, not a lazy wrapper
          component={() => (
            <Suspense fallback={<ActivityIndicator />}> 
              <SettingsScreen />
            </Suspense>
          )}
        />
      </Stack.Navigator>
    </NavigationContainer>
  );
}

Takeaway

Wrap rarely‑used screens in React.lazy + Suspense and register them as a component factory. The bundle shrinks, the app launches faster, and the user only pays the load cost when they actually navigate to that screen.
