Background Tasks in Expo: Keep Work Running When App Is Closed

📅 2026-06-23
Learn how to schedule reliable background work in Expo apps using expo-task-manager and expo-background-fetch, without ejecting to bare workflow.
expo background task-manager

Insight

Expo provides a fully managed way to run code when your app is in the background or even terminated. By pairing expo-task-manager with expo-background-fetch, you can schedule periodic jobs—like syncing data, refreshing a token, or cleaning caches—without leaving the managed workflow. The system decides the exact timing based on OS heuristics, but you can influence it with minimumInterval and stopOnTerminate flags.

Example

import * as TaskManager from 'expo-task-manager';
import * as BackgroundFetch from 'expo-background-fetch';

const FETCH_TASK = 'background-sync';

TaskManager.defineTask(FETCH_TASK, async () => {
  // Your background work here
  await syncPendingChanges();
  return BackgroundFetch.Result.NewData;
});

await BackgroundFetch.registerTaskAsync(FETCH_TASK, {
  minimumInterval: 15 * 60, // 15 minutes
  stopOnTerminate: false,
  startOnBoot: true,
});

Takeaway

Use Expo's background task APIs to offload periodic work—keep your app responsive and your data fresh—while staying in the managed environment. Remember to test on real devices; simulators don’t trigger background fetch reliably.
