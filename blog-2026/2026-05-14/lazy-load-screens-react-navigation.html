Lazy‑Load Screens in React Navigation with Dynamic Imports

📅 2026-05-14
Speed up initial bundle size by loading heavy screens only when needed using React.lazy and React Navigation's lazy option.
react native navigation performance

Insight

When a React Native app grows, the initial JavaScript bundle can become a bottleneck, especially on low‑end devices. While Metro already supports inline requires, you can gain extra mileage by lazy‑loading entire screens. React Navigation’s lazy prop combined with React.lazy lets you defer loading of rarely‑visited routes until the user navigates to them, cutting startup time and memory usage.

Example

// navigation.tsx
import { createNativeStackNavigator } from '@react-navigation/native-stack';
import { Suspense } from 'react';
import LoadingScreen from './LoadingScreen';

const Stack = createNativeStackNavigator();

const SettingsScreen = React.lazy(() => import('./screens/SettingsScreen'));

export default function AppNavigator() {
  return (
    <Stack.Navigator>
      <Stack.Screen name="Home" component={HomeScreen} />
      <Stack.Screen
        name="Settings"
        // Defer loading until navigation occurs
        component={() => (
          <Suspense fallback={<LoadingScreen />}>
            <SettingsScreen />
          </Suspense>
        )}
        options={{ lazy: true }}
      />
    </Stack.Navigator>
  );
}

Takeaway

Enable lazy on each navigator and wrap lazily imported screens with Suspense. This pattern drops the initial bundle size by the screen’s code size and improves perceived launch speed without sacrificing type safety or navigation experience.
