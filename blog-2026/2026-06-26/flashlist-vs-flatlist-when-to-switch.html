FlashList vs FlatList: When to Switch for Massive Lists

📅 2026-06-26
A quick comparison of FlashList and FlatList, showing when FlashList's virtualization wins for huge data sets and how to migrate with minimal code changes.
react native performance flashlist

Insight

FlatList is the default go‑to for scrollable lists in React Native, but its internal recycling algorithm can choke on tens of thousands of items, leading to frame drops and long mount times. FlashList (by Shopify) replaces FlatList's measurement step with a highly optimized, native‑backed layout engine that pre‑computes item dimensions and batches renders more aggressively. The result is smoother scrolling and lower memory pressure, especially when each row is simple and has a predictable height.

Example

import { FlashList } from '@shopify/flash-list';

const data = Array.from({ length: 50000 }, (_, i) => ({ id: i, title: `Item ${i}` }));

export default function MassiveList() {
  return (
    <FlashList
      data={data}
      renderItem={({ item }) => <Text>{item.title}</Text>}
      estimatedItemSize={50} // crucial for performance
    />
  );
}

The only required change from FlatList is swapping the component and providing an estimatedItemSize.

Takeaway

If your list exceeds ~5 k items or you notice jitter during fast scrolls, try FlashList. Its drop‑in API means you can benchmark quickly, and the performance gains often outweigh the extra dependency.
