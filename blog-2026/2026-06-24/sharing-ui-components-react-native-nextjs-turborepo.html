Sharing UI Components Across React Native and Next.js with Turborepo

📅 2026-06-24
Learn how a Turborepo monorepo can let you write a single UI component once and reuse it in both React Native and Next.js, cutting duplication and keeping styles consistent.
monorepo react-native next.js

Insight

A Turborepo monorepo lets you place shared UI code in a common package and compile it for both React Native (via react-native and expo) and the web (via react-native-web). The key is to export components that rely only on the core React Native primitives and avoid platform‑specific APIs. When you import the same component in a Next.js page, the bundler swaps the native implementations for their web equivalents automatically.

Example

// packages/ui/src/Button.tsx
import { TouchableOpacity, Text, StyleSheet } from 'react-native';

export const Button = ({ title, onPress }: { title: string; onPress: () => void }) => (
  <TouchableOpacity style={styles.btn} onPress={onPress}>
    <Text style={styles.text}>{title}</Text>
  </TouchableOpacity>
);

const styles = StyleSheet.create({
  btn: { padding: 12, backgroundColor: '#0066ff', borderRadius: 4 },
  text: { color: '#fff', textAlign: 'center' },
});

Takeaway

Put platform‑agnostic UI in a shared package, configure Turborepo’s build pipeline to transpile with tsc and babel, and you’ll get a single source of truth for components that work everywhere—saving time and preventing drift between mobile and web codebases.
